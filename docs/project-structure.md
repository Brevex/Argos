# Project Structure

Argos keeps one implementation path per technical concern. Source code lives in `src/`, frontend code lives in `frontend/src/`, integration and property-oriented tests live in `tests/`, hot-path benchmarks live in `benches/`, fuzz targets live in `fuzz/`, release resources live in `resources/`, and architecture or algorithm rationale lives in `docs/` and `.opencode/instructions/`.

The repository does not track generated build outputs, dependency installs, IDE state, or local agent runtime state. `target/`, `frontend/dist/`, every `node_modules/` directory, `.agents/`, `.codex/`, and local `.opencode` package installs are disposable local artifacts.

Tracked lockfiles are part of the supply-chain boundary. `Cargo.lock` and `frontend/package-lock.json` remain versioned and must be updated intentionally with dependency changes.

Tauri schema snapshots under `gen/` are generated by the Tauri build tooling. They remain tracked while the build refreshes them deterministically.
